# Инструкции для всех агентов Antigravity

В этом репозитории настроен автоматический CI/CD пайплайн (GitHub Actions) для сборки релизов под Windows и macOS, а также собственный Serverless Proxy сервер для скачивания приватных релизов.

**ПРИ РАБОТЕ С ЭТИМ РЕПОЗИТОРИЕМ СТРОГО СОБЛЮДАЙТЕ СЛЕДУЮЩИЕ ПРАВИЛА:**

1. **НИКОГДА не пытайтесь собирать релизы локально** с помощью команд вроде `npm run build:mac`, `npm run build:win` или `electron-builder`. 
2. **Как выпускать новую версию:**
   - Сделайте нужные изменения в коде.
   - Поднимите версию в `package.json` (например, с `1.0.58` на `1.0.59`).
   - Закоммитьте изменения: `git commit -am "Описание"`.
   - Создайте тег: `git tag v1.0.59` (строго с префиксом `v`).
   - Отправьте тег в репозиторий: `git push origin main --tags` (или `git push origin v1.0.59`).
   - Дальше GitHub Actions сам параллельно соберет Mac (.dmg) и Windows (.exe) версии и выложит их в Releases.

3. **Обновления:** 
   Код проверки обновлений (`electron/main/index.ts`) специально настроен на работу через прокси-сервер Vercel (`https://updater-proxy-rodjer1.vercel.app`). Не меняйте эту логику на прямое обращение к `api.github.com`, иначе пользователи без токенов получат ошибку 404 (так как репозиторий приватный).
